IDE

The KIT library does not require or enforce any specific IDE, i.e. you can use your IDE or editor of choice. That said there are optional scripts/functionality that is IDE specific. For example:

  • Helper scripts to launch the Segger's Ozone debugger
  • Support for VS Code's Clang intellisense and auto-formatting.

If you contribute the KIT library - it is recommended that you use VS Code.

OSAL Implementations

The KIT library's OSAL layer has been ported to run on the following platforms/OS

Operating System Description/Notes
Bare Metal No underlying Operating System and the application is a 'super loop'
POSIX Any POSIX compliant operating system. Note: It has only been validated on Linux (with some limited testing with MacOS)
Windows Windows 10 or higher operating system
FreeRTOS Real-Time Operating System (RTOS) for Microcontrollers. Support for multi-core (SMP) is planned but not yet implemented

Porting Guide - Step-by-step instructions for porting KIT's OSAL.
