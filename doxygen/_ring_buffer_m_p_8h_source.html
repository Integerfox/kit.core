23/** This template class extends the RingBuffer class to be thread safe AND
24 has Uint32 Model Point that tracks the number of elements in the buffer. This
25 give the RingBuffer change notification semantics with elements are added
26 or removed.
27
28 NOTE: The default behavior of the RingBuffer is to initialize the memory
29 buffer to all zeros. When the 'ITEM' type is a class type (i.e. not
30 struct or primitive type) then the 'initializeMemory' constructor flag
31 MUST be set to false - so as to not overwrite the vtable pointer of
32 the class instance.
33
34 Template Args:
35 ITEM:= Type of the data stored in the Ring Buffer
36 */

45 {
46 }
