31/** This concrete class implements a simple "dynamic storage" IString Type.
32 All memory is allocated from the heap. For memory allocation errors,
33 the following happens:
34 1) The _truncated flag is set to true.
35
36 2) If the error occurred in the constructor, then the internal string
37 is set an empty string. If the error occurred because of a requested
38 size increase, the internal string is simply truncated.
39 */
40class DString : public StringBase
41{
42public:
43 /** Constructor. The amount of storage initially allocated for the string is
44 the maximum of the size of the source string AND the value of initialSize.
45 Whatever value is chosen, it is then rounded up to the nearest block size
46 multiple. There are two main reasons to specify an initialSize.
47 1) To reduce the number of malloc/free that occur over the life of the
48 IString. If you supply a large enough initialSize - no additional
49 mallocs will be needed.
50 2) If you plan to use the format(..) methods. The format() methods will
51 NOT allocate additional storage. To prevent the format() methods from
52 truncating, you must start with a large enough 'buffer'.
53
54 The block size parameter can be used to control the size of the 'chunks'
55 memory is allocated in. This 'blocking' paradigm helps to reduce
56 fragmentation and number of internal malloc/free operations.
57
58 NOTE: The space reserved for the trailing null terminator is part of the
59 block calculation, i.e. if your initial size is 16, and your
60 block size is 16, then number of bytes allocated is 32 to allow
61 space for a 16 byte string and a one byte terminator and then
62 rounded up to the next block size.
63 */
64 DString( const IString& string, int initialSize = 0, int blocksize = OPTION_KIT_TEXT_DSTRING_ALLOC_BLOCK_SIZE ) noexcept;
