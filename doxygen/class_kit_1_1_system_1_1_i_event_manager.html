Detailed Description

This abstract class defines the interface to signal a 'Event Flag'.

Event Flags are used to indicate that an 'event' has occurred

Each EventLoop supports up to 32 individual Event Flags (numbered 0-31).

Event Flags are used to indicate that an 'event' has occurred. Individual event flags can be viewed as binary semaphores with respect to be signaled/ waiting (though waiting is done on the EventLoop's entire set of event flags).

A EventLoop can wait for at least one event to be signaled. When the EventLoop is waiting on event(s) and it is then signaled - all of the Events that were in the signaled state when the EventLoop was unblocked are cleared.

The IEventManager interface is inherently thread safe.
