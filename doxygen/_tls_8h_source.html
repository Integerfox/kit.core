39/** This concrete interface defines a interface for Thread Local Storage (TLS).
40 TLS provides a mechanism for each thread to have its own instance of a
41 global variable. A canonical example of TLS is the C error code variable
42 'errno'.
43
44 The implementation assumes no OS support for TLS. However, that said, it
45 can be used on platform where TLS is natively supported, i.e. one less
46 interface to port for a new OS/RTOS.
47
48 NOTES:
49
50 o The initial contents (per thread) of a TLS variable will be zero.
51 o If a new TLS variable/index can NOT be created (i.e. exceeded the
52 Platforms supported limits) a Fatal Error is generated.
53 o A TLS variable can ONLY store a void pointer (or a size_t integer).
54 o Each instance of this class represents a single TLS variable that
55 is SAME across ALL threads, i.e. do NOT create a TLS instance per
56 thread, create a TLS instance per global variable.
57 o TLS instances can NOT be created statically, i.e. they must be
58 created AFTER the Kit::System::initialize() is called.
59 o TLS instances can NOT be copied.
60 o It is PLATFORM SPECIFIC BEHAVIOR if deleting a TLS instance frees
61 up memory for another instance.
62 */
