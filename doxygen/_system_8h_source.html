68/** This concrete class defines the interface for handling and manipulating
69 entries in the System's File system.
70
71 The directory separator is ALWAYS 'dirSep'. If necessary it will be
72 internally converted to the OS/platform specific delimiter. Directory paths
73 can contain drive designators, but ONLY if the native OS supports it.
74 */

327 /** This method breaks the specified File System Entry name into it's
328 individual components. If the specific component is not
329 need, then specify nullptr as the argument and it will be skipped.
330 The fullPath argument contains the drive+path. Path only contains
331 the path. The fullName argument contains the file name+extension.
332 Name only contains the file name. On return, fullPath string (if
333 specified and drive exists) will contain the driver separator. The
334 drive string will NOT 'include' the drive separator. The fullName
335 parameter will contain the extensionSeparator. The name and
336 extension will NOT 'include' the extensionSeparator.
337
338 Note: 'fullPath' and 'path' WILL contain the trailing directory
339 separator.
340 */
