68/** This mostly abstract class defines the operations that can be performed on
69 a thread.
70
71 NOTE: The thread contains a built in counting semaphore that is usable by
72 the developer/application. This semaphore can be used to assist in
73 implementing inter-thread-communications and/or ISR-to-thread
74 communications.
75
76 NOTE: There are default implementations provided for several member and class
77 methods. They are contained in the Thread.cpp file in this directory.
78 If your Operating System port requires a different implementation, then
79 omit compiling the Thread.cpp file (in this directory) and provided
80 your own implementation.
81 */

88 /** This method returns the task's unique identifier. The returned identifier

91 NOTE: The returned value can NOT be used a handle/reference/pointer

93 */
94 virtual KitSystemThreadID_T getId() const noexcept;

96 /** This method returns the 'executing' state of the thread. If the method
97 returns false, the underlying thread has terminated (i.e. the IRunnable

99 can be safely deleted using the destroy() method below.
100
101 NOTE: Default implementation is provided in the Thread.cpp file.
102 */
103 virtual bool isActive() const noexcept;

105 /** This method returns a reference to the thread's IRunnable object

119 /** Special version of getCurrent(). getCurrent() will trigger a FATAL error

126 static Thread* tryGetCurrent() noexcept;

128 /** This method causes the CURRENT thread to wait until its 'thread

151 static bool isActiveThread( Thread* threadPtr ) noexcept;

192 /** Internal Iterator that allows the Client to traverse the list
193 active threads.
194
195 NOTES:
196 o There is difference between an "active" thread and a Thread object
197 instance. The underlying platform/OS determines if the a thread is
198 "active", i.e. the thread is created and running. The Thread object
199 gets created BEFORE the thread is active and exists AFTER thread
200 terminates/end. Why the explanation, this method only includes
201 "active" threads. Threads that have terminated, but that their
202 associated Thread object instance has not been delete/destroyed
203 will NOT show up in the active list.
204 o This call is mutex protected. It will prevent new threads from
205 being created/deleted until it has completed.
206 o A C++ iterator was purposefully not used because access to the list
207 of threads has to be mutex protected. By using an
208 custom-iterator-with-a-callback approach - the Thread class can
209 enforce proper mutex/critical-section handling.
210
211 NOTE: Default implementation is provided in the Thread.cpp file.
212 */
213 static void traverse( Thread::ITraverser& client ) noexcept;

217 /** This method creates a thread according to the specified parameters. The
218 thread/task begins execution immediately upon its creation.
219
220 NOTE: Not all platform support all options, e.g. The C++11 multi threading
221 support does not directly support thread priorities.
222
223
224 @returns A pointer to the Thread created, OR 0 if the was an error.
225
226 @param runnable a runnable object which is started when the task is created
227 @param name a null terminated string that represent that thread's name
228 @param priority a priority ranging from highest priority (KIT_SYSTEM_THREAD_PRIORITY_HIGHEST)
229 to lowest priority (KIT_SYSTEM_THREAD_PRIORITY_LOWEST).
230 Priorities are defined in relative 'levels' to the max/normal/min
231 settings because there is no standardized priority settings
232 across Operating Systems. If the priority value exceeds the
233 upper/lower bounds the platform's priority range, the thread's
234 priority will be clamped to the upper/lower bound of the priority
235 range.
236 @param stackSize Requested size, in bytes, of the stack for the thread. A value of
237 of zero will size the thread's stack per the platform's default
238 stack size setting.
239 @param stackPtr Pointer to the stack memory to be used for the thread's stack. If
240 zero is passed, then the underlying platform/OS will allocate the
241 stack memory.
242 @param allowSimTicks With this argument is true the thread will be created to
243 use/support simulated time when 'Simulate ticks' are enabled.
244 A value false will force the thread to always be a 'real time'
245 thread. See Cpl::System::SimTicks for more details about
246 simulated time.
247 */
248 static Thread* create( IRunnable& runnable,
249 const char* name,
250 int priority = KIT_SYSTEM_THREAD_PRIORITY_NORMAL,
251 int stackSize = 0,
252 void* stackPtr = 0,
253 bool allowSimTicks = true ) noexcept;

256 /** This method is used to destroy a thread that was created by the Thread
257 Factory. If you are going to delete a thread that was created by the
258 Thread Factory, you MUST use the Factory's destroy method to delete it
259 to properly insure that the memory used to create the task is released
260 correctly.
261
262 When the 'delayTimeMsToWaitIfNotStopped' argument is NOT zero, AND the
263 thread is still active. This method will call pleaseStop() on the IRunnable
264 object associated with the thread. After the delay time (in milliseconds)
265 has expired - will attempt to unconditionally terminate the thread.
266
267 NOTES:
268
269 o The application should only delete/destroy a thread AFTER the
270 thread's IRunnable object.run() method has ended/terminated. If
271 the thread is destroy/delete before run() has completed, there is
272 NO GUARANTEE with respect to whether or not the thread has
273 released all acquired resources!
274 o NOT ALL Platforms support brute-force termination of threads,
275 e.g. in C++11 there is no method/interface to forcibly terminate
276 a thread. Once again -->it is a HIGHLY DISCOURAGED to design
277 your application where you need to forcibly terminate threads.
278 o This method only deletes/destroys the Thread instance -- it does
279 NOT delete/destroy the associated IRunnable instance.
280 */
281 static void destroy( Thread& threadToDestroy, uint32_t delayTimeMsToWaitIfActive = 0 ) noexcept;
