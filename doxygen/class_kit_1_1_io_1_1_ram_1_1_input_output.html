Detailed Description

This concrete class implements an InputOutput stream using a Ring Buffer.

NOTES:

  • Thread safe and read and write operations have blocking semantics when the RingBuffer is empty or full, respectively.
  • For concurrent access, there can only be at a single ACTIVE reader and a single ACTIVE writer of the stream. In addition, the concurrent active reader/writer must be in different threads. For serialized access there is no threading restrictions on the readers and writers.
  • This is NOT shared memory across processes, i.e. assumes a SINGLE address space.
  • Implementation is not necessarily efficient - it is more focused on clarity and robustness.
