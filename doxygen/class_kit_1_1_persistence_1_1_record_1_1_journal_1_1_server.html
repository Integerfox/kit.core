Detailed Description

template<class ENTRY>
class Kit::Persistence::Record::Journal::Server< ENTRY >

This concrete template class implements the ITC messaging and Model Point monitoring needed to provide a thread-safe/asynchronous interface for Journal Records.

A single Server instance manages a single EntryRecord instance. Multiple EntryRecords can be managed by creating multiple Server instances

NOTE: The EntryRecords and HeadRecords are NOT managed by the Kit::Persistence::Record::Server. They are wholly managed by this class (Kit::Persistence::Record::Journal::Server)

Template Args: ENTRY:= The Application specific class that defines an 'entry'
