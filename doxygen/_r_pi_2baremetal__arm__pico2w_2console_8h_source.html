13 By default the BSP does NOT provide a KIT IO Stream for application' console.
14 The application can enabled the IO Stream by defining the macro
15 USE_BSP_KIT_IO_STREAM. When the macro is NOT used the C's library's printf()
16 output will be send to the UART directly and the KIT IO stream (g_bspConsoleStream)
17 will not be available.
18
19 NOTE: When USE_BSP_KIT_IO_STREAM is NOT defined, the C library buffers the printf
20 output, i.e. output is not sent until a newline is encountered the output stream
21
22 NOTE: Since the BSP is for a bare-metal platform, there are no threads,
23 which means blocking-wait semantics become BUSY-WAIT semantics.
24 Use the console IO stream with care! Always call available() before
25 reading data.
26
27 The underlying KIT IO Stream is indirectly exposed/available in the global
28 namespace as g_bspConsoleStream. To get access, you code must include the
29 following statement:
30 \code
31 extern Kit::Io::IInputOutput& g_bspConsoleStream;
32 \endcode
33
34 NOTE: Exposing the console stream avoids the BSP header files have a C++ code
