25/** This concrete class implements an InputOutput stream using a Ring
26 Buffer.
27
28 NOTES:
29 - Thread safe and read and write operations have blocking semantics when
30 the RingBuffer is empty or full, respectively.
31 - For concurrent access, there can only be at a single ACTIVE reader and a
32 single ACTIVE writer of the stream. In addition, the concurrent active
33 reader/writer must be in different threads. For serialized access there
34 is no threading restrictions on the readers and writers.
35 - This is NOT shared memory across processes, i.e. assumes a SINGLE
36 address space.
37 - Implementation is not necessarily efficient - it is more focused on
38 clarity and robustness.
39 */
40
41class InputOutput : public Kit::Io::IInputOutput
42{
43public:
44 /** Constructor. 'N' is the number of bytes in 'memoryBuffer'. The actual
45 number of bytes that can be stored in the Ring Buffer is N-1.
46 */

48 : m_ringBuffer( memoryBuffer, N, initializeMemory )
49 , m_readerWaiting( false )
50 , m_writerWaiting( false )
51 , m_started( true )

53 }
