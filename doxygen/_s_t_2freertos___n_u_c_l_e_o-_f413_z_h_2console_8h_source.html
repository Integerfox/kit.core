13 By default the BSP provides a KIT IO Stream for application' console. The
14 application can disabled the IO Stream by defining the macro
15 USE_BSP_USE_PRINTF. When the macro is used the C's library's printf()
16 output will be send to the UART directly and the KIT IO stream (g_bspConsoleStream)
17 will not be available.
18
19 NOTE: When USE_BSP_USE_PRINTF, the C library buffers the printf output, i.e.
20 output is not sent until a newline is encountered the output stream
21
22 The underlying KIT IO Stream is indirectly exposed/available in the global
23 namespace as g_bspConsoleStream. To get access, you code must include the
24 following statement:
25 \code
26 extern Kit::Io::IInputOutput& g_bspConsoleStream;
27 \endcode
28
29 NOTE: Exposing the console stream avoids the BSP header files have a C++ code
