21/** This mostly concrete class implements a "fixed storage" String Type.
22 This class is built on the assumption that "behind-the-scenes"
23 dynamic memory allocation is a bad thing (for a long list of reasons)!
24 Therefore, NO memory allocation is done in this base class. The sub-class
25 is responsible for allocating ALL of storage used by the FStringBase.
26
27 NOTE: All methods that "modify" the string guarantee that the internal
28 storage limit is not exceeded and the string is always NULL
29 terminated. The "incoming" data is silently truncated if the data
30 size exceeds the storage limit.
31 */
