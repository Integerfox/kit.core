Detailed Description

This concrete mutex class defines the interface for a mutex that has "recursive" semantics.

Recursive semantic allows the thread that owns the mutex to acquire ownership multiple times by calling lock() multiple times.
There must be a corresponding number of unlock() calls made before the ownership of the mutex is released.
