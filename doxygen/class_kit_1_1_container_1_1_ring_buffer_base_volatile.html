Detailed Description

This 'implementation' class provides the base functionality for a ring buffer.

The Ring buffer is ISR/Thread safe as long as following constraints are met:

  • Writing to an 'unsigned' data type is an atomic machine instruction.
  • There is SINGLE producer (add) and a SINGLE consumer (remove)

NOTE: This class relies on the 'volatile' qualifier to ensure that the read/write indexes are properly updated across threads/ISRs. It should ONLY be used with hardware targets that DO NOT support std::atomic with full lock-free semantics. For example - the Arm Cortex-M0 does NOT have hardware instructions to support full lock-free atomics.

NOTE: This class is not intended to be used directly. A child class is required to provide the ring buffer memory and type safety.
