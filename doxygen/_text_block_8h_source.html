21/** This concrete class tokenizes a Text Block that has the following
22 format: <pre>
23
24 parameter [(delimiter parameter)*] [terminator]
25
26 parameter := { all printable characters except for QUOTE, DEL, TERM, and ESC }
27 QUOTE := { specified quote character (used to start/end TEXT STRINGS) }
28 DEL := { specified delimiter character }
29 TERM := { specified terminator character }
30 ESC := { specified Escape character used inside of TEXT STRINGS }
31
32 The tokenizer replaces the delimiter and terminator characters with an '\0'
33 character. The tokenizer supports the parameter fields being "text strings".
34 In addition, any leading and trailing whitespace is removed from each
35 parameter. Note: no whitespace is altered/removed from the contents of a
36 text string.
37
38 If the string being parsed has two DEL characters in row, then tokenized
39 results will return an 'empty parameter' (a parameter will a have zero
40 string length) for this sequence. The EXCEPTION to this rule is when the
41 delimiter is the space character, then the tokenizer will never generate
42 and/or detect any 'empty parameters'.
43
44 The original string is "corrupted" since the tokenizer changes characters.
45 Also, the tokenizer will shorten the string when dealing with 'text
46 strings'.
47
48 A TEXT STRING can contain any alphanumeric and/or punctuation characters
49 including the QUOTE, DEL, TERM, and ESC characters. A text string is
50 always enclosed with beginning and ending QUOTE characters. When a QUOTE
51 character is encounter outside of a text string, it always starts a text
52 string. To enter a literal QUOTE character within a text string, precede the
53 QUOTE character with the ESC character (e.g., \"). To enter a literal ESC
54 character within a text string, precede the character with ESC character
55 (e.g., \\‍). IMPORTANT NOTE: The QUOTE and ESC characters are REMOVED from
56 the string when it is tokenized!
57
58
59</pre>*/
