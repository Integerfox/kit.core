21/** This macro defines the integer type (aka the size of) of the boot counter.

23 over. If your application has very frequent reboots, you may want to
24 increase the size of the boot counter type. However, the more bits allocated
25 to the boot counter - less bits are available for elapsed milliseconds.

30 */

47/** This method returns 'Boot Time' which include the current boot counter
48 and elapsed time - in milliseconds - since the MCU was reset.
49
50 The method retuns boot time as 'flatten' uint64_t integer where the
51 upper N bytes are the boot counter, the lower M bytes are elapsed MCU time
52 since reset.
53
54 NOTE: The default implementation uses the Kit::System::ElapsedTime interface
55 as the source for elapsed milliseconds. The implementation can be
56 changed/overridden by defining USE_KIT_TIME_BOOTTIME_WITH_ABSOLUTETIME

58 used for the elapsed time portion of the boot time.
59 */
