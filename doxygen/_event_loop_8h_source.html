33/** This concrete class is a IRunnable object that provides a event driven
34 execution model for a thread. The thread will remaining blocked until
35 an "event" occurs. The following "events" are supported:
36
37 1) The EventLoop Semaphore was signaled (this is separate from the Thread Semaphore)
38 2) A Event Flag was signaled
39 3) A Software Timer expiring
40
41 To receive a callbacks when Event Flags are signaled, the application must
42 provide a list of Event Flag instances to the Event Loop constructor.
43
44 Child classes can be used to extend the 'event processing' to include other
45 types of events, e.g. Inter-Thread-Communication messaging.
46
47 NOTE: The EventLoop does NOT use/consume the Thread Semaphore.
48 */
49class EventLoop : public IRunnable, public IEventManager, public ISignable, public TimerManager
50{
51public:
52 /** Constructor. The 'timeOutPeriodInMsec' parameter specifies how
53 long the EventLoop will wait for an event before timing out and
54 processing the software timers. The value of 'timeOutPeriodInMsec' is
55 the resolution of the timing source for the Software Timers, i.e. the
56 event loop will 'wake-up' at least every 'timeOutPeriodInMsec'
57 milliseconds.
58
59 A fatal error is generated if 'timeOutPeriodInMsec' is set to zero.
60
61 When the 'eventFlagsList' parameter is null then the EventLoop will not
62 monitor any event flags.
63
64 The 'watchdog' parameter is optional and can be used to provide
65 watchdog monitoring for this event loop. If used, the application
66 must also enable the Watchdog using the USE_KIT_SYSTEM_WATCHDOG
67 configuration option in its the kit_config.h header files
68
69 WARNING: The application CANNOT modify the eventFlagsList after the
70 EventLoop instance is created.
71 */
72 EventLoop( uint32_t timeOutPeriodInMsec = OPTION_KIT_SYSTEM_EVENT_LOOP_TIMEOUT_PERIOD,
73 Kit::Container::SList<IEventFlag>* eventFlagsList = nullptr,
74 IWatchedEventLoop* watchdog = nullptr ) noexcept;
75
76 /// Virtual destructor
77 virtual ~EventLoop() = default;
78

80protected:
81 /** This method is used to initialize the Event Loop's thread has started
82 to executed.
83
84 This method is intended to be used by child classes that are extending
85 the Event Loop. For this use case - this method MUST be called once on
86 the beginning of the entry() method and BEFORE the "main loop" for the
87 entry() method is entered.
88 */
89 virtual void startEventLoop() noexcept;
90
91 /** This method is used to wait (and process) the next event(s). This
92 method returns after being woken up and processing all timer expired
93 and pending Event Flag events. This method should always be wrapped
94 in a loop (typically a forever loop).
95
96 If the 'skipWait' argument is set to true then the method does not
97 wait for event - it simply processing the current event flags and
98 timers. The use scenario for skipping the wait is for child classes
99 to execute the event loop multiple times to drain their message/notifications
100 queues one message/notification at a time (i.e. give equal time/priority
101 to all types of events)
102
103 The method typically returns true. The method only returns false if
104 the pleaseStop() method was called on the Event Loop instance.
105
106 This method is intended to be used by child classes that are extending
107 the Event Loop. For this use case - this method MUST be called inside
108 the child class's "main loop" in its the appRun() method.
109
110 Example appRun() for a child class extending the Event Loop:
111 @code
112
113 void ChildEventLoop::entry()
114 {
115 startEventLoop();
116 <child specific initialization>
117
118 bool run = true;
119 while( run )
120 {
121 run = waitAndProcessEvents();
122 if ( run )
123 {
124 <child specific event processing>
125 }
126 }
127 stopEventLoop()
128 }
129
130 @endcode
131 */
132 virtual bool waitAndProcessEvents( bool skipWait = false ) noexcept;
133
134 /** This method is used to clean-up the Event Loop's when the thread is
135 being stopped.
136
137 This method is intended to be used by child classes that are extending
138 the Event Loop. For this use case - this method MUST be called once
139 AFTER the event-processing loop has exited.
140 */
141 virtual void stopEventLoop() noexcept;
