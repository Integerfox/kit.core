Detailed Description

This partial concrete class provides common implementations, utility methods, etc.

for concrete child classes that implement the IChunk interface. The class ASSUMES the following:

  • All child classes will use a checksum to detect corrupted data.
  • There is sufficiently large temporary buffer to hold a Record's data plus the additional meta data persistent stored. Note: The private_.h header file provides a default buffer. See private_.h for additional details about the default buffer.
  • Some of helper methods (e.g. pushToRecord() and pullFromRecord()) are virtual wrapper methods. By having a virtual wrapper - pushing/pulling to/from the data record facilitates future child classes to extend existing chunk classes with the ability to decrypt/encrypt the record data

IMedia & media )
protectednoexcept

Helper method to read a Size_T value from the persistence storage.

It has the following behavior/side-effects:

  • The 'value' is accumulated into the CRC calculation (i.e. m_crc.accumulate() is called)
  • The 'offset' is updated to point to the next byte after the Size_T value that was read
  • The default behavior is to use m_workBuffer as the buffer to read the Size_T value into RAM. However, a different buffer can be provided via the workBuffer parameter.

Returns true if the read was successful, else false is returned.

bool Kit::Persistence::Record::Chunk::Common::writeSizeT ( Size_T value,
Size_T & offset,
IMedia & media )
protectednoexcept

Helper method to write a Size_T value to persistent storage.

It has the following behavior/side-effects:

  • Converts the 'value' from the Host/MCU Endianess to the media's Endianess before writing to persistent storage
  • The 'value' is accumulated into the CRC calculation (i.e. m_crc.accumulate() is called). The accumulation is done AFTER any endianess conversion is done.
  • The 'offset' is updated to point to the next byte after the Size_T value that was written

Returns true if the write was successful, else false is returned.
