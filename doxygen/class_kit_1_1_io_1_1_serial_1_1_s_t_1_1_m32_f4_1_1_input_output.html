Detailed Description

This concrete class implements the InputOutput stream interface built on top of ST HAL Layer/SDK.

The implementation ASSUMES that at most there is only ONE client attempt to use the InputOutput stream at any given time. It is okay to call the read(), write(), etc.from different threads - but the calls CANNOT be concurrent. It is the application's responsibility to provide additional thread-safety/concurrence protection.

Note: The 'receiver' and the 'transmitter' methods MAY be called concurrently.

This method MUST be used to finish the initialization of the stream and start is underlying drivers.

The application is responsible for fully initializing 'uartHdlToUse' BEFORE calling this method.
